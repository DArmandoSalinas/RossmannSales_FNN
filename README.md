[![TensorFlow](https://img.shields.io/badge/TensorFlow-2.15%2B-orange.svg)](https://www.tensorflow.org/)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

End-to-end deep learning pipeline for the [Rossmann Store Sales](https://www.kaggle.com/c/rossmann-store-sales) Kaggle competition. A feedforward neural network (MLP) forecasts daily sales across 1,115 German drugstores using engineered tabular features, dropout regularization, and early stopping.

**Author:** [Diego Armando Salinas Lugo](https://sites.google.com/tec.mx/salinasdiegoarmando/know-me)

---

## Highlights

- Reproducible preprocessing with missing-value handling and temporal feature engineering
- 3-layer MLP (128 → 64 → 32) with ReLU activations, dropout, and Adam optimizer
- Train/validation split with feature and target scaling
- Kaggle-ready `submission.csv` export

## Quick start

### 1. Clone and install

```bash
git clone https://github.com/DArmandoSalinas/RossmannSales_FNN.git
cd RossmannSales_FNN
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Download data

1. Accept the competition rules on [Kaggle](https://www.kaggle.com/c/rossmann-store-sales/data).
2. Download `train.csv`, `test.csv`, and `store.csv`.
3. Place them in `data/raw/`.

### 3. Preprocess

```bash
python -m src.preprocess
```

### 4. Train and generate predictions

```bash
python -m src.train
```

The submission file is written to `outputs/submission.csv`.

### Optional: explore in Jupyter

```bash
jupyter notebook notebooks/ExploringTrainingTesting.ipynb
```

## Model architecture

| Layer | Units | Activation | Notes |
|-------|-------|------------|-------|
| Input | 8 features | — | Scaled tabular features |
| Dense | 128 | ReLU | + Dropout (0.3) |
| Dense | 64 | ReLU | + Dropout (0.3) |
| Dense | 32 | ReLU | |
| Output | 1 | Linear | Daily sales forecast |

**Training:** MSE loss · Adam (lr=1e-4) · batch size 32 · up to 100 epochs · early stopping (patience=10)

**Features:** `Open`, `Promo`, `Promo2Open`, `DayOfWeek`, `CompetitionDistance`, `CompetitionOpen`, `Month`, `DayOfYear`

## Preprocessing pipeline

- Impute store-level missing values (median distance, zero for unknown promo/competition dates)
- Parse dates and extract calendar features (year, month, week, day-of-year)
- Merge store metadata into train/test sets
- Compute `CompetitionOpen` and `Promo2Open` duration in months
- Drop leakage-prone columns (`Customers`, raw date fields)

## Results

The model captures non-linear relationships between promotions, competition, seasonality, and sales. Validation RMSE is printed after training; upload `outputs/submission.csv` to Kaggle for official leaderboard scoring (RMSPE metric).
